Documentation installation infrastructure réseau - Configuration client


Configuration réseau d’un client

Windows :
Par default Windows devrait reconnaitre le pare-feu comme passerelle. Nous n’avons donc rien de spécial à configurer.
Linux :
Par default Ubuntu n’identifie pas notre pare-feu comme passerelle par default pour accéder à internet. Ainsi nous devons la configurer manuellement. Malheureusement, étant donné que nous utilisons le DHCP sur nos machines clients, configuré la passerelle par défaut est légèrement plus compliquée qu’avec le serveur. Pour se faire nous pouvons utiliser la commande suivante :
« ip route add default via »
Seul bémol de cette commande est qu’il faudra la retaper à chaque démarrage de l’ordinateur
Pour éviter ce problème deux solution existe :
Premièrement vous pouvez automatiser l’exécution de la commande à chaque démarrage du poste de travail via crontab. Vous devrez donc ajouter la commande suivante à votre crontab : « @reboot « ip route add default »
La seconde solution est celle que nous avons choisis :
Nous devons modifier le ficher « /etc/NetworkManager/system-connections/Wired connection 1.nmconnection » pour y indiquer notre nouvelle passerelle par défaut :

Puis nous rajoutons la ligne suivante à la fin de la partie [ipv4] du fichier : « route1=0.0.0.0/0,,100 »


Après avoir redémarré le poste de travail nous pouvons vérifier que la route a correctement été ajouté grâce à la commande « ip route »


Ici nous remarquons bien la première ligne qui correspond à l’ip de notre pare-feu .
